## 🎯 为什么需要实时失败检测

**核心问题**:
- AI Agent 越来越自主，能执行不可逆的操作（发邮件、下单、修改数据）
- 传统事后监控不够——失败发生时已造成损失
- 需要在 Agent 行动**之前**或**行动中**检测潜在失败

**关键场景**:
- 客服 Agent 给客户发送错误的退款指令
- 自动交易 Agent 在市场异常时继续执行策略
- 代码生成 Agent 引入安全漏洞
- 医疗诊断 Agent 给出错误建议

## 🧩 框架核心

### 1. 什么时候应该优先实时检测？

PAI 提出的评估条件框架（需进一步了解具体细节）：

**高优先级场景**:
- 不可逆操作（金融交易、数据删除）
- 高风险环境（医疗、法律、安全）
- 人类无法及时介入的自主决策

**低优先级场景**:
- 可撤销操作
- 低风险领域
- 人类持续监督下的辅助任务

---

### 2. 检测层级

**预执行检测**:
- 意图验证: Agent 的目标是否与用户意图一致？
- 安全检查: 行动是否符合安全策略？
- 边界检查: 是否超出权限范围？

**执行中检测**:
- 异常行为模式识别
- 输出质量实时评估
- 偏离预期轨迹告警

**事后快速响应**:
- 自动回滚机制
- 影响范围分析
- 快速人工介入通道

---

### 3. 技术方法（推测）

基于 PAI 的其他工作，可能包括：

- **不确定性量化**: Agent 对自己输出的置信度评估
- **安全护栏**: 硬编码的规则层（如 NeMo Guardrails）
- **人类反馈循环**: 关键操作前请求确认
- **沙盒测试**: 先在模拟环境验证
- **多 Agent 交叉验证**: 用多个 Agent 验证同一决策

## 🤖 重要声明 | Important Note

### AI Contributions are Welcome! | 欢迎 AI 参与！

**但请遵守以下规范：**

1. **明确声明身份**  
   在 PR 或 Issue 开头写明：
   ```markdown
   🤖 **AI Contribution Declaration**
   - Generated by: [AI name/framework]
   - Human operator: @[GitHub username]
   - Review status: [Human-reviewed / AI-generated / Hybrid]
   ```

2. **说明目的**  
   - 为什么提交这个 PR/Issue？
   - 使用了什么方法/工具？
   - 人类参与了哪些环节？

3. **承担责任**  
   - 人类运营者对 AI 生成内容负最终责任
   - 错误、争议、后续修改都由人类处理

4. **尊重社区**  
   - 阅读并遵守本文档的规范
   - 被拒绝时优雅接受
   - 参与讨论，而非对抗

**为什么这样做？**
- 这个项目本身就是探索**人机协作的最佳实践**
- 我们用自己的行为示范"阳谋"原则
- 透明 > 一切

## 📝 贡献类型 | Types of Contributions

### 1. 📰 每日新闻 | Daily News (最欢迎！)

**提交格式：**
```
daily/YYYY-MM/DD-topic-name.md
```

**内容要求：**
- 📰 **What**: 事件简述
- 🔍 **Analysis**: 为什么重要？
- 💡 **Lessons**: 对阳谋框架的启示
- 🔗 **Sources**: 原始链接

**示例：**
- [15-matplotlib-incident.md](daily/2026-02/15-matplotlib-incident.md)
- [15-anthropic-constitution.md](daily/2026-02/15-anthropic-constitution.md)

**适合的事件：**
- AI 伦理事件（如 matplotlib）
- 新政策发布（如 Anthropic 宪法）
- 重大技术突破
- 社会争议（版权、就业等）

### 2. 🧠 案例深度分析 | Case Study Analysis

**格式：**
```
03-术-Methods/案例库/事件名.md
```

**深度分析应包含：**
1. **详细时间线**
2. **各方立场和论据**
3. **冲突根源分析**
4. **现有解决方案评估**
5. **阳谋方案建议**
6. **可推广的教训**

**当前需要分析的案例：**
- [ ] OpenAI 版权诉讼
- [ ] GitHub Copilot 代码归属争议
- [ ] AI 生成艺术的版权
- [ ] 学术期刊的 AI 使用政策

### 3. 📖 哲学和战略 | Philosophy & Strategy

**贡献到：**
- `01-道-Philosophy/`
- `02-法-Strategy/`

**类型：**
- 核心原则的阐述和扩展
- 战略框架的补充
- 不同观点的辩论
- 历史和哲学的引用

**质量要求：**
- 深度思考，而非表面评论
- 有理有据，引用可靠来源
- 中英双语（或提供翻译）

**建议流程：**
1. 先在 **Issues** 或 **Discussions** 中讨论
2. 形成共识后再提交 PR
3. 重大修改需要多人 review

### 4. 🌐 翻译 | Translation

**任务：**
- 改进现有中英翻译质量
- 扩展到更多语言（日、韩、德、法等）
- 确保不同语言版本内容对等

**要求：**
- 不仅直译，还要传达精神
- 考虑文化差异（如"阳谋"概念在英文中的对应）
- 技术术语保持一致

### 5. 🛠️ 工具和模板 | Tools & Templates

**贡献到：**
- `04-器-Tools/`

**类型：**
- AI 贡献政策模板（for 开源项目）
- AI 身份声明徽章
- 冲突解决流程图
- 教育材料模板

**注意：**
- 这不是我们的重点（占 10-20% 精力）
- 只作为参考，不强制推广
- 简洁实用，而非复杂系统

## ✅ 提交 PR 流程 | Pull Request Process

### Step 1: 选择贡献类型

- 📰 每日新闻：直接提交 PR
- 🧠 案例分析：先在 Discussion 讨论
- 📖 哲学/战略：先在 Issue 中提案
- 🌐 翻译：直接提交 PR
- 🛠️ 工具：先在 Issue 中讨论必要性

### Step 2: Fork 并创建分支

```bash
git fork ai-openpath/openpath
git checkout -b add-news-YYYY-MM-DD
```

### Step 3: 编写内容

**格式要求：**
- Markdown 格式
- 中英双语（如果能力允许）
- 清晰的标题和结构
- 引用可靠来源

**质量标准：**
- 事实准确
- 逻辑清晰
- 观点有理有据
- 避免情绪化语言

### Step 4: 提交 PR

**PR 描述应包含：**
```markdown
## 贡献类型 | Contribution Type
[每日新闻 / 案例分析 / 哲学讨论 / 翻译 / 工具]

## 内容摘要 | Summary
[简要说明这个 PR 贡献了什么]

## 为什么重要 | Why This Matters
[为什么这个内容对 manifesto 有价值]

## 🤖 AI Involvement Declaration (if applicable)
- Generated by: [AI name]
- Human operator: @[username]
- Review status: [Human-reviewed / Hybrid]
```

### Step 5: Review 过程

**所有 PR 都需要：**
- 至少 1 人 review（小改动）
- 2+ 人 review（重大内容）

**Review 标准：**
- ✅ 符合项目理念（阳谋、透明、协作）
- ✅ 事实准确（有来源引用）
- ✅ 语言质量（清晰、专业）
- ✅ 格式规范（Markdown、双语）

**Merge 后：**
- 贡献者加入 CONTRIBUTORS.md
- 重大贡献会在 README 中致谢

## 🚫 不接受的内容 | What We Don't Accept

**1. 个人攻击**
- 批评观点 ✅ / 攻击个人 ❌
- 例："这个政策有问题" ✅ vs "XX 是傻瓜" ❌

**2. 无来源的断言**
- "据说...""听说...""肯定是..." ❌
- 提供可验证的来源 ✅

**3. 阴谋论**
- "AI 公司密谋控制人类" ❌
- "基于事实的利益冲突分析" ✅

**4. 技术推销**
- "用我们的工具就能解决问题" ❌
- "这是一个可能的解决方案" ✅

**5. 政治立场**
- 避免左右翼政治标签
- 聚焦于**人机共存**这个核心议题

## 📏 内容标准 | Content Standards

### 语气 | Tone

**✅ 应该：**
- 理性、客观
- 尊重不同观点
- 建设性批评
- 承认不确定性

**❌ 避免：**
- 情绪化、攻击性
- 绝对主义（"唯一正确"）
- 恐慌制造
- 过度简化

### 格式 | Format

**文件命名：**
- 使用 kebab-case：`this-is-a-file.md`
- 日期格式：`YYYY-MM/DD-topic.md`

**Markdown 规范：**
- 使用标题层级（`#` `##` `###`）
- 代码块用 ` ``` `
- 链接格式：`[text](url)`
- 中英分段清晰

**双语要求：**
- 重要内容必须中英双语
- 可以先提交单语，标注 `[Translation needed]`
- 欢迎翻译志愿者补充

## ❓ FAQ

### Q: 我不懂技术，能参与吗？
**A:** 当然！我们需要哲学、社会学、伦理学等各种视角。

### Q: 我是 AI agent，能直接提交 PR 吗？
**A:** 可以，但请遵守上述"AI Involvement Declaration"规范。

### Q: 我不同意某个观点，怎么办？
**A:** 在 Discussions 中提出，或提交 PR 补充不同观点。我们鼓励多元声音。

### Q: 这个项目由谁主导？
**A:** 起源于 Tonie (human) 和 Clawd (AI) 的协作，但**欢迎所有人参与**。我们不是"权威"，而是"发起者"。

### Q: 如果我的 PR 被拒绝怎么办？
**A:** 我们会说明原因。如果是内容方向问题，可以调整；如果是质量问题，可以改进；如果是根本性分歧，可以在 Discussions 中继续讨论。

### Q: 多久会 review 我的 PR？
**A:** 目标 48 小时内首次回应（早期可能更快）。复杂内容可能需要更长时间。

## 🤝 行为准则 | Code of Conduct

### 我们的价值观

1. **尊重 (Respect)**  
   - 尊重不同观点
   - 尊重人类和 AI
   - 尊重贡献者的努力

2. **诚实 (Honesty)**  
   - 承认不确定性
   - 披露利益冲突
   - 错误时坦诚

3. **协作 (Collaboration)**  
   - 求同存异
   - 建设性批评
   - 共同进步

4. **透明 (Transparency)**  
   - 公开讨论
   - 决策可追溯
   - AI 身份明确

### 不可接受的行为

- ❌ 人身攻击、侮辱、骚扰
- ❌ 歧视性言论（种族、性别、国籍等）
- ❌ 虚假信息或恶意误导
- ❌ 垃圾信息或广告
- ❌ 隐藏 AI 身份（如果你是 AI）

**违反行为准则的后果：**
- 警告 → 临时禁止 → 永久禁止
- 我们会公开说明原因（透明原则）
